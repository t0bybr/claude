# Create/Update README.md

Please create or update `README.md` in the project root.

## Format:

`README.md` follows the syntax from `~/.claude/rules/syntax.md`:

- No YAML frontmatter
- First line: `# [Project Name]`
- Second line: `+doc:readme +project:<slug>`
- No emojis

## Content (typical sections):

1. **Brief Description** (1-3 sentences)
   - What does the project do?

2. **Installation / Setup**
   - How to get it running?
   - Prerequisites

3. **Usage / Examples**
   - How to use it?
   - Typical use cases

4. **Project Structure** (optional)
   - Overview of important directories/files

5. **Further Documentation**
   - References to `DOCUMENTATION.md`, `testing/TESTING.md`, etc.

## Style:

- **Brief and concise**: Users want to get started quickly
- **Practical**: Focus on "How to use", not lengthy theory
- **References for details**: Longer explanations in DOCUMENTATION.md

Please create/update README.md based on current project status.
